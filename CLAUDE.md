# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project

Kan — open-source Trello alternative (kan.bn). pnpm + Turborepo monorepo, AGPLv3. Node >= 20.18.1, pnpm 9.14.2.

## Commands

All commands run from the repo root. Env vars live in a single root `.env` (copy from `.env.example`); package scripts load it via `dotenv -e ../../.env`.

```bash
pnpm dev              # run everything (turbo watch)
pnpm dev:next         # web app + its deps only
pnpm build            # turbo run build
pnpm lint / lint:fix
pnpm format / format:fix
pnpm typecheck
pnpm db:migrate       # drizzle-kit migrate (packages/db)
pnpm db:push          # drizzle-kit push
pnpm db:studio
```

Tests use Vitest, only in `apps/web` and `packages/api` (no root test task):

```bash
pnpm --filter @kan/api test                                    # one package
pnpm --filter @kan/api exec vitest run src/routers/board-move.test.ts   # single file
pnpm --filter @kan/web test:watch                              # watch mode
```

i18n (Lingui, in apps/web): `pnpm --filter @kan/web lingui:extract` then `lingui:compile`. UI strings use Lingui macros; locale catalogs are in `apps/web/src/locales/`.

**Adding an env var:** it must also be listed in `turbo.json` `globalEnv`, or turbo won't pass it through.

## Architecture

Request flow: **Next.js page → tRPC client → router (`packages/api`) → repository (`packages/db`) → Drizzle/Postgres**.

- `apps/web` — Next.js 15 **pages router** (not app router), React 18, Tailwind, Headless UI. Structure: `src/pages/` are thin route files; the actual page UI lives in `src/views/<feature>/` (each with its own `components/`); shared UI in `src/components/`. Client-side env via `next-runtime-env`, validated in `src/env.ts`.
- `packages/api` — tRPC v11 routers in `src/routers/`, composed in `src/root.ts`. **Dual transport:** the same routers are exposed as tRPC (`/api/trpc`) and as a public REST API (`/api/v1`) generated by `trpc-to-openapi` from each procedure's `.meta({ openapi: ... })` — new procedures should include that meta plus `.input()`/`.output()` zod schemas. Use `protectedProcedure` (auth) and `assertPermission(db, userId, workspaceId, "…")` from `src/utils/permissions.ts` for authorization.
- `packages/db` — Drizzle ORM. Tables in `src/schema/`, all data access through function-per-operation repositories in `src/repository/*.repo.ts` (routers never touch Drizzle directly). Also exports the Redis client (`./redis`, used for rate limiting).
- `packages/auth` — Better Auth config (email/password, magic link, many social providers, OIDC, Stripe plugin). Mounted at `apps/web/src/pages/api/auth/[...all].ts`.
- `packages/email` — React Email templates sent via SMTP.
- `packages/stripe`, `packages/logger` (pino), `packages/shared` (utils like `generateUID`), `packages/mcp` (published `@kan/mcp` MCP server that talks to the REST API), `apps/docs` (Mintlify docs).
- `tooling/*` — shared eslint/prettier/tailwind/tsconfig presets, consumed as `@kan/eslint-config` etc.

### Conventions

- **publicId vs id:** every entity has an internal numeric `id` (never exposed) and a 12-char `publicId` (`generateUID()` from `@kan/shared/utils`) used in all API inputs/outputs and URLs. Repos resolve publicId → id.
- Soft deletes: rows get `deletedAt`/`deletedBy`; queries filter `isNull(deletedAt)`.
- Non-web API routes (Stripe webhooks, uploads, Trello import auth) live in `apps/web/src/pages/api/`.
